  1. 필요에 따라 다음 필수 구성요소를 설치하십시오.

    C++ 컴파일러 C++ 11을 지원해야 합니다. GCC 6.0 이전 버전은 이 이슈에 의해 동작하지 않습니다. maxOS에서는 XCode로 잘 동작합니다.
    Go 1.10 이상의 버전이 필요합니다.
    Bash 4 버전 이상이 좋습니다. 하지만 3.x 에서도 동작하는 것으로 알려저 있습니다.
    CMake 3.81 이상 버전이 동작하는 것으로 알려저 있습니다.
    Autoconf 2.68 이상의 버전이 필요합니다.

    A 64-bit system is strongly recommended. Building or running CockroachDB on 32-bit systems has not been tested. You'll also need at least 2GB of RAM. If you plan to run our test suite, you'll need closer to 4GB of RAM.

  2. Download the CockroachDB {{ page.release_info.version }} source archive, and extract the sources:

    copy
    $ curl https://binaries.cockroachdb.com/cockroach-{{ page.release_info.version }}.src.tgz | tar -xJ

  3. In the extracted directory, run make build:

    {% include copy-clipboard.html %}
    cd cockroach-{{ page.release_info.version }}
    {% include copy-clipboard.html %}
    make build

    The build process can take 10+ minutes, so please be patient.

    {{site.data.alerts.callout_info}}The default binary contains core open-source functionality covered by the Apache License 2 (APL2) and enterprise functionality covered by the CockroachDB Community License (CCL). To build a pure open-source (APL2) version excluding enterprise functionality, use make buildoss. See this blog post for more details.{{site.data.alerts.end}}

  4. Install the cockroach binary into /usr/local/bin so it's easy to execute cockroach commands from any directory:

    {% include copy-clipboard.html %}
    make install

    If you get a permissions error, prefix the command with sudo.

    You can also execute the cockroach binary directly from its built location, ./src/github.com/cockroachdb/cockroach/cockroach, but the rest of the documentation assumes you have the binary on your PATH.

  5. Keep up-to-date with CockroachDB releases and best practices:

    C++ compiler C++ 11을 지원해야 합니다. GCC 6.0 이전 버전은 이 이슈에 의해 동작하지 않습니다. maxOS에서는 XCode로 잘 동작합니다.
    Go 1.10 이상의 버전이 필요합니다.
    Bash 4 버전 이상이 좋습니다. 하지만 3.x 에서도 동작하는 것으로 알려저 있습니다.

    Terminfo / NCurses 개발 라이브러리 이것은 ncurses 개발 패키지의(데비안/우분투의 libncurses-dev 또는 CentOS의 curses-devel) 일부입니다.

  2. CockroachDB {{ page.release_info.version }} 아카이브를 다운로드하고 압축을 푸십시오.

    copy
    $ wget -qO- https://binaries.cockroachdb.com/cockroach-{{page.release_info.version}}.src.tgz | tar  xvz

  3. 압축이 풀린 디렉토리로 이동한 후,make build를 실행하십시오.

    {% include copy-clipboard.html %}
    cd cockroach-{{ page.release_info.version }}
    {% include copy-clipboard.html %}
    make build

    빌드는 10분 이상 걸릴수 있습니다. 기다려 주십시오.

    {{site.data.alerts.callout_info}} 기본 바이너리는 Apache License 2 (APL2)를 가진 core 오픈소스 기능과 CockroachDB Community License (CCL)를 가진 엔터프라이즈 기능을 포함하고 있습니다. 엔터프라이즈 기능을 제외한 순수 오픈소스 버전 (APL2)을 빌드하려면 make buildoss를 사용하십시오. 이 블로그 글에서 세부내용을 확인하실 수 있습니다.{{site.data.alerts.end}}

  4. cockroach 바이너리를 /usr/local/bin에 설치하면 아무 곳에서나 cockroach 명령어를 사용할 수 있습니다.

    {% include copy-clipboard.html %}
    make install

    건한이 없다면 sudo를 사용하십시오.

    cockroach 바이너리를 빌드된 곳에서 사용할 수 있습니다. ./src/github.com/cockroachdb/cockroach/cockroach, 하지만 문서의 나머지 부분은 바이너리가 PATH에 있다고 가정합니다.
